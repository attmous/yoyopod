YoyoPod — Home Screen · Implementation Spec

Canonical mockup of the Home screen: the landing surface the device returns to on long-press from any category root. Five states are documented — idle (no slot focused) and one focused state per deck slot (Listen, Talk, Ask, Setup). The companion Blob lives only on Home; no other screen renders it. Panel canvas is 240 × 280 px.

2 · Screen composition

Home is the simplest composition of all screens — just three persistent regions plus the Blob. No Card, no List, no Tabs.

Layer (z)ElementBounds (x,y,w,h)Notes
z=0 Panel (0, 0, 240, 280) Root lv_obj, bg bg_cream.
z=30 StatusBar (0, 0, 240, 22) Persistent across all screens.
z=1 HomeStage (0, 22, 240, 206) Empty cream surface. Hosts only the Blob.
z=10 Blob (65, 70, 110, 110) Anchor: stage-centered. Y = 48 within stage = 70 in panel coords.
z=20 Deck (0, 228, 240, 52) Persistent across all screens.

Object budget: 1 panel + 1 statusbar (~8 children) + 1 stage + 1 blob (1 canvas) + 1 deck (4 slot children + glyph labels + 1 shared focus pill) = ~17 objects. The cheapest screen on the device.

Flat pass: the deck has no top rule and no slot dividers. The focused slot renders an inset accent pill — fill inset 6 px vertical / 8 px horizontal (60 × 52 slot → 44 × 40 pill), radius 12 — behind the glyph. Idle slots are bare ink_300 glyphs on the cream band. Glyph box 30 px (Unicode reference here; A8 icon assets at runtime), centered, identical in both states; only glyph color and pill fill swap. Pill + glyph transition is 160 ms ease — no relayout, no borders. Home's deck slots do not use the system focus ring (2 px ink outline + scale 1.1, reserved for actionable controls) — the pill fill carries focus.

Escape hatch: if the LCD washes out fill-on-fill (accent pill on cream), darken the stage tint or add a 1 px self-color outline to the pill — a token change, not a layout change.

6 · Input contract (Home only)

EventFrom idleFrom focused state N
Single press FocusFirst → focus = Listen FocusNext → focus = (N+1) mod 4 (Listen → Talk → Ask → Setup → Listen)
Double press no-op (no focused target) Open → push category N's depth-1 root
Long press no-op (already at Home) defocus (return to idle)
Idle timeout — after 8 s of no input from a focused state, return to idle (defocus)
Ambient timeoutafter ~30 s of no input on idle Home, enter Ambient: deck fades out, Blob keeps the full stage (presence state, see mockup_input_model.html §3)— (focus decays to idle first)
Any input in Ambientwake to Home idle; deck fades/slides back in over 200 ms

Home is the only screen where the focusable container is the Deck. On every other screen it's the List (or NowPlaying controls).

Voice prompt on focus (pre-reader bridge): every focus advance speaks the newly-focused slot's name through the speaker ("Listen", "Talk", …). Applies to all focusable containers on all screens. Toggle: Setup → "Speak names". The full gesture grammar and its hardware bindings (main button today, whole-surface touch later) live in mockup_input_model.html — the single source of truth.

Deck lv_obj flex row 4 × deck_slot + 1 focus pill bg=bg_cream. border_side NONE — top rule removed in flat pass.
Deck slot lv_obj + label glyph label No borders — slot dividers removed in flat pass. Focus = accent pill lv_obj (44 × 40, radius 12, inset 6/8) moved to the focused slot + glyph color swap (style swap, no relayout); pill hidden on idle.

If sprite-atlas memory is tight, the Blob can be drawn vector-style with three lv_objs (body via radius animation, two eyes, one mouth arc) and a per-frame lv_anim driving radius + scale. Sprite path is preferred — fewer style recalculations, less CPU.

9 · Behavior rules (quick reference)

  • Stage color is fixed at bg_cream on Home regardless of focused slot. (Per-category stage tints are reserved for inside-category screens only.)
  • Blob color is fixed at c_lime — it's the device's identity, not a category mirror.
  • Idle state is the canonical landing: long-press from any depth-1 category root returns here and clears deck focus.
  • Deck focus wraps: Listen → Talk → Ask → Setup → Listen. There is no backward step; long-press defocuses instead.
  • No tabs, no card, no list on Home. If you need to render any of those, you're not on Home.
  • Status bar is independent of focus state — never changes color, contents, or position.

10 · Asset checklist for implementers

  • Blob sprite atlas: PNG-8 or RGB565 sprite sheet, frame size 110 × 110, alpha for the character outline edge (drop shadow removed in flat pass — no shadow alpha needed). Recommend 32 frames (~135 KB at RGB565).
  • Deck glyphs: A8 icon assets at runtime, tinted ink_300 (idle) / ink (focus); the Unicode glyphs here (🎧︎ 💬︎ ? ⚙) are design-time reference only.
  • StatusBar icons (signal bars, GPS pin, battery): procedural draw — no asset files needed.
  • Verify RGB565 LUT in assets/theme.ron matches Section 8 values exactly.
  • If the Blob is animated procedurally (not sprite), expose a single boolean blob_enabled to allow disabling animation on low-battery — saves ~3% CPU at 30 FPS.
