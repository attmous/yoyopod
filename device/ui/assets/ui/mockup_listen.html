YoyoPod — Listen Flow · v2 · Wheel + Arc Hero · Implementation Spec

Canonical mockup of the Listen category, redesigned around two primitives: the wheel (carousel menu — the focused tile is the only boxed element on screen) and the arc hero (ring-progress NowPlaying). Tab strip, card, and rows are retired. This file owns both primitives — mockup_talk.html references them. Every pixel in the renders is sourced from the §3/§4 tables — no magic numbers. Panel canvas is 240 × 280 px: status bar y 0–22 · stage y 22–228 · deck y 228–280. One hardware button (press = roll · double-press = open · long-press ≥ 400 ms = Home) + PTT side button; a voice prompt speaks every focused label.

1 · Screen ladder — wheel depth 1 → 3, roll, arc hero

Listen root → Playlists → Morning Songs tracks → NowPlaying, plus a mid-roll animation frame and the two NowPlaying variants (paused, composed fallback). Stage tints lime across the whole flow because the focused deck slot is Listen. Glyphs in these renders are page approximations — at runtime each maps to an A8 icon asset (§12).

Behavior

InputResult
Press Roll one step, down-to-up: SelectionOffset 0 → −80, 180 ms ease-out cubic, wraps at the ends. TTS speaks the new focused label at t = 0 (interruptible by the next press).
Double-press Open the focused item — 220 ms push transition.
Long-press ≥ 400 msGo Home (pop the whole stack). Playback state unaffected.

Edge cases

Item countWheel behavior
1 No roll — press plays a nudge (offset 0 → 12 → 0, 120 ms) and re-speaks the label.
2 Single peek below only (the next-in-wrap item); an item is never duplicated above and below simultaneously.
3–6Standard 3-slot window with wrap; peek above = previous-in-wrap, peek below = next-in-wrap.
> 6recycle_window: 3 — only 3 (4 mid-roll) item objects live; the context label appends a counter, e.g. PLAYLISTS · 4/12.

6 · Component → LVGL map

ComponentCanonical stock widgetComposed with today's FFIVerdict
Wheel lv_roller — REJECTED: text-only, cannot host icons / plates / sub-labels. DeckKind::List restyle: lv_obj slots + lv_image + lv_label, ScaleOnFocus{700,1000}, selected_roles chrome, SelectionOffset roll, recycle_window: 3. Composed IS canonical.
Progress ring lv_arc — new FFI + ElementKind::Arc (§7). Bar fallback = existing Progress emulation (track obj + fill obj), frame 7. lv_arc canonical; bar ships until the FFI lands.
Hard offset shadow lv_obj_set_style_shadow_offset_x/_y FFI (§7). Sibling ink rect at +3,+3 behind the element — zero FFI, +1 obj per shadowed element. Either; FFI preferred (fewer objects).
Peek scaling lv_obj_set_style_transform_scale (or lv_image_set_scale) FFI (§7). Pre-baked 40×40 icon variants + opacity only (labels stay full-size). FFI preferred; baked assets acceptable interim.
Focus outline lv_obj_set_style_outline_color (+ width, pad) FFI (§7). Border 2 → 3 px on the focused transport target. FFI preferred; border fallback documented per target.
Chips / buttons lv_obj with radius = h/2 + lv_image glyph — same on both paths. lv_button unnecessary.

10 · Input contract

EventOn wheelOn arc heroVoice prompt
Single press Roll one step down-to-up (wraps). Cycle focus prev → play → next (wraps; default play). New focused label at t = 0 — "Sleep" / "Next song".
Double press Open the focused item (220 ms push). Activate the focused control (toggle play/pause, skip). Destination label — "Morning Songs"; on hero: "Playing" / "Paused".
Long press ≥ 400 ms Go Home (pop the whole stack). Go Home — playback continues. "Home".

The PTT side button is Talk-domain and unchanged — full gesture contract in mockup_input_model.html. Runtime divergence (carried from v1): the shipped OneButtonMachine (input/machine.rs) emits Back on long-hold and core pops one screen — remap to pop-to-root when implementing.
