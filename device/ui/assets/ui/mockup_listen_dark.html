YoyoPod — Listen Flow · v2 · Wheel + Arc Hero · Dark Theme

Dark-theme variant of the canonical Listen spec (mockup_listen.html · "Listen Flow · v2.1", flat wheel + arc hero). Strict token swap: page structure, geometry, behavior, and renders are identical to the light file — only the surface/ink color tokens change, plus one dark-only token, ink_on_accent, for content sitting on a bright accent fill. Chrome stays flat, color-first (v2.1): no borders, no shadows — the one surviving stroke is the transient focus ring on transport targets, drawn in dark ink (warm off-white). The light file is normative for everything except color — every pixel in the renders below is sourced from its §3/§4 tables. Panel canvas is 240 × 280 px: status bar y 0–22 · stage y 22–228 · deck y 228–280. One hardware button (press = roll · double-press = open · long-press ≥ 400 ms = Home) + PTT side button; a voice prompt speaks every focused label.

5 · Typography

Identical to mockup_listen.html §5 — the theme does not touch fonts. Montserrat only: 12 / 18 linked, 14 / 24 enabled in lv_conf.h but pending the FFI extern link (light §7). The runtime has no emoji font — every glyph in the renders maps to an A8 icon asset (§12).

6 · Component → LVGL map

Identical to mockup_listen.html §6 — wheel = composed DeckKind::List restyle (canonical), progress ring = lv_arc (bar fallback ships until the FFI lands), flat chrome = border_width 0 everywhere. The theme switch is a token swap at theme.ron load — no widget or composition changes. The only dark addition is resolving ink_on_accent as the content color of accent-filled roles (in light that field can simply default to ink — same hex).

7 · Minimal FFI additions

Single source: mockup_listen.html §7 — the redesign's FFI wishlist is not duplicated here. Dark adds no FFI: ink_on_accent is a plain color token, and v2.1 (flat) already deleted the shadow FFI rows (so v1-dark's shadow_ink plumbing is moot).

9 · Animation & motion

Identical to mockup_listen.html §9 — wheel roll 180 ms ease-out cubic, push/pop 220 ms, arc lerp ≤ 1000 ms linear, deck focus 160 ms, nudge 120 ms. The only dark difference is the color endpoints of the deck-focus stage-tint crossfade, which use the dark bg_stage_* values from §2. The SelectionOffset → Y remap flag (lvgl_renderer.rs:157) applies unchanged.

10 · Input contract & voice

Identical to mockup_listen.html §10 — single press rolls / cycles focus, double press opens / activates, long-press ≥ 400 ms goes Home; every focused label is spoken at t = 0. Voice prompts are theme-independent. PTT is Talk-domain (see mockup_input_model.html). The OneButtonMachine long-hold divergence (emits Back, pops one screen — remap to pop-to-root) applies unchanged.

11 · Object budget

Identical to mockup_listen.html §11 — wheel worst case 46, arc hero 33, composed fallback ~34, all under the 60-object cap with ≥ 14 headroom. The token swap adds zero objects: ink_on_accent is a style value, and dark has no shadow rects for the same reason light doesn't (flat).

12 · Asset checklist for implementers

  • No new bitmaps for dark. Every icon is A8 (alpha-only) and recolored per theme at draw time — the same assets serve both themes (wheel icons 56×56, ♪ note glyph, transport 24×24, deck glyphs).
  • RGB565 dark LUT in theme.ron must match §2 exactly — this regeneration corrected bg_stage_butter (0x29E3 → 0x2923) from v1-dark.
  • The silent icon_key → SETUP-bitmap fallback warning (light §12, icons.rs) applies unchanged — register every key, add a debug assert.
  • Montserrat 14 / 24 extern FFI links: see light §7 — theme-independent.
