YoyoPod — Listen Flow · Dark Theme

Dark-theme variant of the Listen spec. Geometry, spacing, type, and accents are identical to the light theme — only the surface and ink tokens swap. Accents stay vivid against the deep surfaces; a new ink_on_accent token (near-black) carries text/borders that sit on top of an accent fill (focused row, active tab, focused deck slot, play button, cover art glyph). Panel canvas is 240 × 280 px.

11 · NavStack contract

struct Crumb {
    name:   &'static str,
    accent: AccentColor,   // captured at push time
}

enum AccentColor { Lime, Peri, Butter, Coral }

// Rendering rules:
//  - TabStrip draws nav.crumbs.iter().take(3)
//  - If nav.crumbs.len() > 3, prepend a "…" chevron tab
//  - The deck's active category is the IMPLICIT root — not stored in crumbs
//  - Leaf screens (NowPlaying, IncomingCall, VoiceNote) DO NOT push their own crumb
//      → active tab stays on the parent layer's name
//  - On push: previous active tab demotes (height 18→14, color c_*→cream_2, opa 1.0→0.85)
//  - On pop:  rightmost tab dies, previous tab promotes back to active

12 · Asset checklist for implementers

  • Font: Inter (weights 600, 700, 800, 900). License OFL — bundle in assets/fonts/.
  • Glyphs (vector or pre-rendered): ♪ (Listen), 💬︎ (Talk), ? (Ask), ⚙ (Setup), ⏮ ▶ ⏸ ⏭ (NowPlaying), ›.
  • Cover-art fallback: 60×60 PNG, c_mustard fill, ink border drawn at render — no asset needed.
  • StatusBar icons: signal bars (procedural), GPS pin (procedural), battery (procedural). No raster assets.
  • RGB565 LUT in assets/theme.ron must match Section 2 values exactly. RGB888 → RGB565 conversion: ((R&0xF8)<<8)|((G&0xFC)<<3)|(B>>3).
