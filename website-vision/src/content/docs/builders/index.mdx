*For engineers and contributors: what yoyopod is made of (hardware), how it
thinks (software), and how to build, flash, and test one.*

## How this guide is organized

Three parts, from atoms to workflows:

1. **Hardware Platform** — the physical device: display, audio, power,
   connectivity, and the path from Pi prototype to a product board.
2. **Software Platform** — the running system: architecture, the device
   runtime and its workers, the UI system, cloud, apps, and security.
3. **Developer Guide** — you and the repo: environment, build & flash,
   testing, and the API/SDK surface.

Vision pages here describe the **target** platform. For what is implemented
today, each page's *Today vs. target* section points to the as-built
engineering docs (the `website/` site in this repository — coral accent),
which include full deep-dive guides for the UI system and the runtime.
