## Suggested build order

1. **Flat theme + deck pill** on existing screens — a pure `theme.ron`
   token pass; immediate visible win, no new machinery.
2. **Wheel on Listen** (items 1–3, 7, 8) — Listen's routes already match
   the shipped router, so this is pure presentation.
3. **Arc hero on NowPlaying** — start with the zero-FFI bar fallback,
   upgrade to `lv_arc` when item 5 lands.
4. **Input remap** (item 4) + voice prompts (item 9).
5. **Talk v5** (item 10), then Ask, then Setup.
6. **System overlays** (item 13) — smallest, do anytime.

## Acceptance checklist (per screen)

- Renders match the spec's device frames at 240×280 — judged against the
  **RGB565 token values**, not the CSS hexes.
- Press / double-press / long-press behave per the screen's input-contract
  table; long-press lands on Home everywhere except capture screens, where
  hold = talk.
- Focus is visible without reading: fill + size (+ ring on controls).
- The focused label is spoken on every focus change (when "Speak names" is
  on).
- No UI borders or shadows anywhere; characters keep their ink outlines.
- Object count stays within the spec's §budget.

## Deliberately open — do not block on these

Notifications on Home without auto-play · stuck-send retry policy · replay
cap (~20/contact suggested) · contact ordering (pinned suggested) ·
double-press window for small hands (350 ms default, to be tested with
kids) · parent gate on Setup · haptics · the LCD washout escape hatch
(darken the stage tint or a 1 px self-color outline — a token change, only
if device testing shows the need).
