This guide is the playbook for the yoyopod UI system: every hardware and
software component involved in presenting any shape or animation on the
canvas, what each layer offers, what it doesn't, and the workarounds in
between. It documents the system as it **is** — including the honest
divergences between shipped code and the design contract. (Its sibling,
the [Runtime & Workers Guide](/runtime/), covers everything behind the
canvas.)

## How this guide is organized

The guide follows the stack **bottom-up**: physical foundations first, then
the software stack layer by layer, then how people interact with it, then
the working references. The Advanced part cuts across all of it.
